## North-star invariants (from W9.4 + W4.6 + W10.4)

These are properties no PR is allowed to break.

- **Never miss an obligation or opportunity.** This is the install/keep
  threshold for the user. Every feature is judged against it.
- **Never auto-send.** Drafts go to Gmail Drafts for review, never to
  Send. (W4.6, S3.3)
- **Never leak confidential or proprietary content** into an LLM
  prompt. PHI, PI, banking info, personal names, SSN, driver licenses,
  non-public business addresses/emails are auto-redacted before any
  LLM call. (W10.1, W10.4)
- **Every LLM call is logged.** Daily-rotated, exportable, Gemini-readable. (W10.5)
- **Predictability over flash.** If using the product adds friction
  without changing the day's organization, the user uninstalls. (W9.3)

## Cross-cutting invariants

(These are the same as "north-star" above, restated as engineering-testable
properties.)

1. The system **never auto-sends** email. Only Gmail Drafts. (W4.6, AD-004)
2. The redaction filter runs **before** every LLM call, with an
   automated test covering all classes in W10.4. (SY-016)
3. **No contact data crosses a contact boundary** in autodraft context
   retrieval, except via explicit link. (SY-008, AD-008)
4. **Every LLM call produces an audit-log row** in the daily Google
   Sheet. No LLM call without a row. (SY-017, AD-005)
5. Sensitive-flagged contacts are excluded from search, digests,
   drafts-for-others, and any LLM context. (US-014, AD-001)
6. The Monday "What matters this week" view contains exactly the five
   swimlanes the exec named — not four, not six. (US-017)
